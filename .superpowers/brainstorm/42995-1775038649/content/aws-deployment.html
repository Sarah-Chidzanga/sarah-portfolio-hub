AWS Console Deployment

Design Section 4 of 4 — Step-by-step through the AWS GUI. No CLI needed.

Step 1 — S3 Bucket (photo storage)
AWS Console → S3 → Create bucket
• Name: sarah-portfolio-photos
• Region: us-east-1 (or closest to you)
• Uncheck "Block all public access" for photos
• Add a bucket policy to allow public reads
Upload sunset photos here — the URL goes into DynamoDB as s3_url.
Step 2 — DynamoDB Tables (×5)
AWS Console → DynamoDB → Create table
Create each table with:
• Partition key: pk (String)
• comments also needs sort key: sk (String)
• Capacity: On-demand (pay per request — free tier friendly)
Repeat for: projects · sunset_photos · books · comments · page_visits
Step 3 — IAM Role for Lambda
AWS Console → IAM → Roles → Create role
• Trusted entity: Lambda
• Attach policies:
— AmazonDynamoDBFullAccess
— AmazonS3ReadOnlyAccess
— AWSLambdaBasicExecutionRole
• Name: sarah-portfolio-lambda-role
Step 4 — Lambda Function
AWS Console → Lambda → Create function
• Runtime: Python 3.12
• Execution role: sarah-portfolio-lambda-role
• Upload code as a .zip file:
zip -r app.zip app.py lambda_handler.py routes/ templates/ static/ db.py requirements/ site-packages/
• Handler: lambda_handler.handler
• Memory: 256 MB · Timeout: 30s
Step 5 — API Gateway
AWS Console → API Gateway → Create API
• Type: HTTP API (simpler + cheaper than REST)
• Integration: Lambda → your function
• Route: ANY /{proxy+} → catches all routes
• Deploy to stage: prod
→ You get a public URL like: https://abc123.execute-api.us-east-1.amazonaws.com
Step 6 — Custom Domain (optional)
AWS Console → Route 53 or API Gateway → Custom domains
• Buy a domain in Route 53 (e.g. sarahchidzanga.com)
• Request an SSL cert in ACM (free)
• Map domain → API Gateway stage
Optional but makes the portfolio feel professional. Can skip for now and add later.
Deployment Order Summary
1. S3 bucket → 2. DynamoDB tables (×5) → 3. IAM role → 4. Lambda function → 5. API Gateway → 6. Custom domain (optional)
Re-deploying after code changes: Zip your updated files → Lambda → Upload from → .zip file. That's it — no CLI needed. We'll include a simple shell script package.sh that creates the zip for you so it's one step.
